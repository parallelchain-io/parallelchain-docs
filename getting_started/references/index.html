    pchain References

    Important

    We recommend that you store pchain in a directory of your choice so that it is easier to follow the commands in the reference.

    For Linux/macOS users - For example, we created a folder in our home directory called parallelchain_client: 

    $ mkdir -p /home/my_user/parallelchain_client
$ cp pchain /home/my_user/parallelchain_client/
$ cd /home/my_user/parallelchain_client
$ ./pchain

    So from now on, when you see a command like this in linux/macOS. It means that pchain shall be executed from the directory you stored pchain in. 

    ./pchain

    For Windows users - Windows uses a different command structure compared to Linux and MacOS. All you need to do is to go to the directory where pchain is located. For example we have a command in PowerShell that shows this: 

    PS> Set-Location C:Users\my_user\parallelchain_client
C:Users\my_user\parallelchain_client> pchain.exe

    So from now on, when you see a command like this in Windows. It means that pchain shall be executed from the directory you stored pchain in. 

    pchain.exe

    Help

    You can always use the flag --help or -h as the last flag for any commands in the following sections to learn more about any built-in commands.

    Command

    ./pchain --help

    pchain.exe --help

    Setup

    Set up configuration variables necessary for VeryLight's operation.

    Command Description
    pchain Check the usage of the light client
    pchain setup networking Configure the IP address of the ParallelChain F Fullnode that VeryLight sends Transactions and queries to.
    pchain setup key-pair Register an Ed25519 KeyPair for VeryLight to use to sign Transactions.

    Networking

    Configure the IP address of the ParallelChain F Fullnode that VeryLight sends Transactions and queries to.

    Command

    • <TARGET_URL>: Configure the URL of the ParallelChain F Fullnode that VeryLight sends Transactions and queries to. (Standard API)
    • <RICH_API_URL>: Configure the URL of the ParallelChain F Fullnode that VeryLight sends rich queries. (Rich API)
    • ANALYTICS_API_URL>: Configure the URL of the ParallelChain F Fullnode that VeryLight sends analytics queries (Analytics API)
    ./pchain setup networking 
    --target_url https://testnet1.digital-transaction.net
    --rich_api_url https://testnet1.digital-transaction.net/api
    --analytics_api_url https://testnet1.digital-transaction.net/analytics

    pchain.exe setup networking --target-address https://testnet1.digital-transaction.net

    Returns

    Returns a success message indicating the target address set.

    Output 
    target_url set.
rich_api_url set.
analytics_api_url set.

    Key Pair

    Register an Ed25519 KeyPair for VeryLight to use to sign Transactions.

    • <KEYPAIR_JSON_PATH>: Absolute path to a 'keypair.json' file. You can generate this file using the Crypto command. Read VeryLight's repository README for the file format.

    Command

    ./pchain setup key-pair --keypair_json_path <KEYPAIR_JSON_PATH>

    pchain.exe setup key-pair --keypair_json_path <KEYPAIR_JSON_PATH>

    Returns

    Returns a success message indicating the keypair json path set.

    Output 
    keypair_json_path set.

    Crypto

    Cryptographic tools for generating key pair and signature.

    Command Description
    pchain crypto generate-key-pair Generate keypair for account creation
    pchain crypto sign Sign a message using VeryLight's registered KeyPair.

    Generate Key Pair

    Register an Ed25519 KeyPair for VeryLight to use to sign Transactions.

    Command

    ./pchain crypto generate-key-pair

    pchain.exe crypto generate-key-pair

    Returns

    Returns a success message indicating the keypair json has been saved.

    Output 
    keypair.json saved in current directory.

    Sign

    Sign a message using VeryLight's registered KeyPair. (If you've not registered a KeyPair, use the Setup command). This prints out the resulting ciphertext in Base64 encoding

    • <MESSAGE>: A message to sign, encoded in Base64.

    Command

    ./pchain crypto sign --message <MESSAGE>

    pchain.exe crypto sign --message <MESSAGE>

    Returns

    Returns a success message indicating the keypair json has been saved.

    Transaction

    Command Description
    pchain submit tx Submit transactions to ParallelChain Blockchain
    pchain query tx Get executed transaction information with transaction hash

    Submit transaction

    You can submit different kinds of transactions using pchain submit tx. The value in each field varies based on its usage. All arguments are required, and the testnet node returns the same acknowledgment message. See the points below on configuring each argument based on your usage:

    • Transfer Token
    • Deploy Contract
    • Call a Contract

    Transfer token

    See "EtoE: Transferring Tokens from One Account to Another" for detailed steps.

    • <FROM_ADDRESS>: the source account address. It is usually your account address.
    • <TO_ADDRESS>: the destination account address. This address must be an Externally Owned Account(EOA).
    • <VALUE>: number in TXPLL. The token amount to transfer
    • <GAS_LIMIT>: Any token transfer between EOAs consumes a fixed amount of gas. We recommend setting this to 44,600.
    • <DATA>: no data is needed. Set this to null
    • <FROM_ACCOUNT_NONCE>: number. You can get the nonce by query command
    • <FROM_ACCOUNT_KEYPAIR>: your account keypair

    Command

    ./pchain submit tx \
--from-address <FROM_ADDRESS> \
--to-address <TO_ADDRESS> \
--value <VALUE> \
--tip <TIP> \
--gas-limit <GAS_LIMIT> \
--gas-price 1 \
--data <DATA> \
--nonce <FROM_ACCOUNT_NONCE> \
--path-to-keypair-json <FROM_ACCOUNT_KEYPAIR>

    pchain.exe submit tx \
--from-address <FROM_ADDRESS> \
--to-address <TO_ADDRESS> \
--value <VALUE> \
--tip <TIP> \
--gas-limit <GAS_LIMIT> \
--gas-price 1 \
--data <DATA> \
--nonce <FROM_ACCOUNT_NONCE> \
--path-to-keypair-json <FROM_ACCOUNT_KEYPAIR>

    Returns

    Returns a successful acknowledgment message from the testnet node, including the transaction hash. Use this hash to verify if your transaction has succeeded. Please see the section on "Query transaction" for details

    Output 
    Hash of tx: "EB9nbVTRFZwxr3tVYmQI8fjawWxW/F0KVe6B1shWGr1="
Status 200
Response "Your request has been received."

    • <FROM_ACCOUNT_NONCE>: number. You can get the nonce by query command
    • <FROM_ACCOUNT_KEYPAIR>: Your account keypair

    Checks the details of a transaction using a transaction hash that is obtained from submitting a transaction.

    World state

    Command Description
    pchain query state Get data from world state with provided key

    Query the state

    You can query data from the world state in a particular block.

    Where:

    • <BLOCK_HASH>: The block's hash for the query.
    • <ADDRESS>: The address of contract account.
    • <KEY>: The key in the world state in format of base64 string.

    Command

    ./pchain query state \
--block-hash <BLOCK_HASH> \
--address <ADDRESS> \
--key <KEY>

    pchain.exe query state \
--block-hash <BLOCK_HASH> \
--address <ADDRESS> \
--key <KEY>

    Returns

    Returns the world state of a particular block

    Example Output The value is in format of base64 string. If key is not found (or not set before), it outputs empty string (decoded to empty array) 
    Your value "AAAAAA=="
Your value(decoded) [0, 0, 0, 0]

    Analyze

    Command Description
    pchain analyze gas-per-block Get the gas usage per block for a specified period
    pchain analyze mempool-size Get mempool size for a specified period

    Gas usage per block

    You can get the gas usage of blocks for a specified period.

    • <START_TIME>: Start of the time window (in unix timestamp formatting)
    • <END_TIME>: End of the time window (in unix timestamp formatting)
    • <WINDOW_SIZE>: Specified period of analysis. (in seconds)
    • <STEP_SIZE>: The amount of time that moves between each window (in seconds)

    Command

    ./pchain analyze gas-per-block \
--from-time <START_TIME> \
--to-time <END_TIME> \
--window-size <WINDOW_SIZE> \
--step-size <STEP_SIZE>

    pchain.exe analyze gas-per-block \
--from-time <START_TIME> \
--to-time <END_TIME> \
--window-size <WINDOW_SIZE> \
--step-size <STEP_SIZE>

    Returns

    Returns the gas usage of a block.

    Example Output The output array contains gas usage value representing period of <WINDOW_SIZE> seconds. The starting time for period is scheduled by every <STEP_SIZE> seconds. The lenght of the array varies depends on the input. 
    Your value [0, 0, 5057700, 44200, 44200, 44200, 44200]

    Mempool size

    You can get the mempool size for a specified period.

    • <START_TIME>: Start of the time window (in unix timestamp formatting)
    • <END_TIME>: End of the time window (in unix timestamp formatting)
    • <WINDOW_SIZE>: Specified period of analysis. (in seconds)
    • <STEP_SIZE>: The amount of time that moves between each window (in seconds)

    Command

    ./pchain analyze mempool-size \
--from-time <START_TIME> \
--to-time <END_TIME> \
--window-size <WINDOW_SIZE> \
--step-size <STEP_SIZE>

    pchain.exe analyze mempool-size \
--from-time <START_TIME> \
--to-time <END_TIME> \
--window-size <WINDOW_SIZE> \
--step-size <STEP_SIZE>

    Returns

    Returns the mempool size for a specified time window

    Example Output The length of the array varies depends on the input. 
    Your value [0, 0, 0, 0, 0, 0, 0]
